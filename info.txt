BoardVault — Project Overview

Summary
BoardVault is a lightweight platform for cataloging, storing, and serving electronic board designs and associated assets. It provides a simple backend API for creating, updating, querying, and deleting board metadata, and stores visual assets (photos, diagrams, pinouts) in Google Drive with controlled public access.

Goals
- Provide a searchable, well-structured index of board metadata and technical specifications.
- Preserve high-quality visual assets while minimizing storage and bandwidth via server-side image compression.
- Keep deployment and integration simple for small teams or hobby projects.

Key features
- Board metadata: structured fields for name, manufacturer, description, tags, technical specs, and timestamps.
- Visual assets: photos and diagrams are uploaded and stored on Google Drive; files are made publicly readable but only after explicit upload and permission creation.
- Simple protection for destructive operations: a password read from `.env` is required for create/update/delete operations.

Operational notes
- Image uploads are compressed server-side to reduce size while retaining sufficient visual fidelity for documentation and reference.
- The backend exposes REST endpoints for CRUD operations and a separate upload endpoint for assets.

Contributing
